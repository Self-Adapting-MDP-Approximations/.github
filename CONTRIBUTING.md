# Contributing to Self-Adapting MDP Approximations

Thank you for your interest in contributing to Self-Adapting MDP Approximations. This project is intended to be a public research-facing tutorial resource for approximate dynamic programming, adaptive methods for Markov decision processes, and reproducible scholarly computation.

We welcome contributions that improve the clarity, quality, usability, and research value of the repository.

## Ways To Contribute

Examples of helpful contributions include:

- reporting bugs or reproducibility problems,
- improving documentation and explanations,
- suggesting corrections or clarifications,
- proposing additional examples or tutorial extensions,
- contributing code improvements that are consistent with the project scope,
- recommending relevant papers or references.

## Before You Contribute

Please first check whether your question or proposal fits one of these channels:

- use GitHub Discussions for conceptual questions, research discussion, and broader ideas,
- use GitHub Issues for bugs, documentation problems, or specific actionable requests,
- use Pull Requests for concrete proposed changes.

If you are planning a substantial change, it is usually best to open an issue or discussion first so the maintainers can help align the proposal with the goals of the project.

## Pull Request Guidelines

When submitting a pull request, please aim to:

- keep the change focused and well-scoped,
- explain the motivation for the change clearly,
- document any important assumptions,
- update relevant documentation when needed,
- preserve readability and tutorial value, not only technical correctness.

For code contributions, please prefer:

- clear and readable implementations,
- comments only where they genuinely improve understanding,
- names and structure that are accessible to future readers,
- changes that do not break existing tutorial workflows without explanation.

## Documentation and Examples

Because this repository is tutorial-oriented, documentation is a core part of the project rather than an afterthought. If a code change makes the repository easier to understand, reproduce, or teach from, that is a meaningful contribution.

We especially value contributions that:

- make advanced ideas easier to follow,
- improve notebook explanations,
- clarify algorithmic steps,
- strengthen the connection between theory and implementation.

## Community Expectations

Please engage with the project in a respectful, constructive, and scholarly way. We welcome thoughtful questions, corrections, and discussion. For community standards, please also read the `CODE_OF_CONDUCT.md`.
